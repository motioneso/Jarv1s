Set: Assistant & AI

This should be about how Jarvis behaves and what AI resources it can use. It should not look like an internal provider/model admin console.

Recommended grouping
Assistant personality
Name, tone, directness, humor level, encouragement/accountability style.
Assistant autonomy
Permission posture, confirm-before-action defaults, what Jarvis may do without asking later.
AI source
Use shared assistant vs personal provider/API key/CLI subscription.
Model capability health
Plain-language readiness: chat ready, summaries ready, vision ready. Hide routing internals.
Advanced provider details
Provider type, credentials, base URL, model IDs. Present as setup/edit flow, not dashboard cards.
Remove from normal settings
Capability lookup widget, assistant tool list, risk strings, raw credential JSON-first UI.

Designer implication

The top of this section should feel like "tune my assistant." Technical AI provider setup can be one advanced path underneath it, because most users should not need to understand model routing.
