Set: Memory & Context

This is a trust surface: what Jarvis remembers, what it believes, and what it is allowed to use.

Recommended grouping
Conversation recall
Toggle whether Jarvis can use past conversations when responding.
Remembered facts
Visible list of facts/preferences/goals/profile notes Jarvis has learned.
Corrections
User can delete, correct, or lower confidence/importance of a memory.
Context categories
Preferences, facts, profile, goals. Human labels, not raw category enums.
Use boundaries
What memory can influence: chat, briefings, task ranking, notifications, planning.
Remove from default UX
Importance sliders as primary UI, sourceThreadId, embedding/index terminology.

Designer implication

This page should feel editable and reversible. It is less "AI settings" and more "inspect and correct what Jarvis thinks it knows."
