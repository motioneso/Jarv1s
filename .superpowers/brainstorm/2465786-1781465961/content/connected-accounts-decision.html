Connected Accounts: Decision

Keep provider setup separate from global data-source controls. Google is one current way to power these sources, not the product concept.

Provider-agnostic model
Accounts
Google now; Microsoft/other providers later. Shows authentication health, reconnect, revoke.
Calendar source
Global Jarvis calendar controls: include in planning, briefings, travel/focus signals, write-back later.
Email source
Global Jarvis email controls: scan for commitments, summarize, include in briefings, sensitivity thresholds.
Provider details
Which account backs each source, scopes granted, last sync, sync health. Details, not top-level IA.

Designer implication

Design around source types the user understands: Calendar, Email, and future data sources. Providers are connection methods underneath each source.
