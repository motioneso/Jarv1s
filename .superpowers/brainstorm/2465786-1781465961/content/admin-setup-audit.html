Admin / Setup Taxonomy

Admin settings should govern the instance. Host/operator setup should be available, but not mixed into the main people/access experience.

Recommended admin grouping
People & access
Users, pending approvals, active/deactivated state, admin roles, revoke sessions.
Identity & registration
Allow registration, require approval, login provider status. Provider config itself is env/setup.
Instance modules
Enable/disable optional modules across the whole instance; required modules locked.
Connector oversight
Safe metadata/health only. No private synced data and no secrets.
Audit & operations
Audit events, data export/delete references, backup/restore status or documentation links.
Advanced host setup
Multiplexer, CLI availability, local runtime checks. Advanced/setup area, not normal admin.

Cut / move from current page

Auth provider status should not be a standalone card. Multiplexer should move to advanced setup. Admin connector metadata should be health/oversight only, not a random compact list.
