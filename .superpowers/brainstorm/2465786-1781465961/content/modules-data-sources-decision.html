Set: Modules & Data Sources

Core settings should show module availability and link outward. Detailed module settings belong to each installed/enabled module, not the default platform settings page.

Admin level
Instance module control
Admin can enable/disable optional modules for the entire instance.
Required modules
Settings, Tasks, AI, Chat, Connectors, etc. may appear as locked/on if required.
Admin visibility
Show module status, required/optional, installed version, and link to module admin settings when present.
User level
Personal module use
User can choose which enabled optional modules they want visible/active for themselves.
Module settings links
Each enabled module can expose its own settings page. Core settings links out.
Tool permissions by module
Assistant tool controls belong inside module settings, not the platform default page.

Designer implication

The default platform settings page is a directory/control center. It should not become one giant form containing every installed module's settings.
