Set: Connected Accounts & Data Sources

Users should see trusted external accounts and what Jarvis can use from them, not backend connector provider records.

Recommended grouping
Connected accounts
Google account identity, status, last sync, needs attention, reconnect, revoke.
Data categories
Calendar, Gmail, contacts/docs later. Show enabled/disabled and why Jarvis uses each.
Permissions / scopes
Human labels for access granted. Raw scopes can be tucked under details.
Use in Jarvis
Toggles for briefing inclusion, task inference, calendar planning, email signal extraction.
Add account
Single action opens a provider picker. Google now; Microsoft/others later.
Remove from default page
Connector Providers card, provider type badges, secret stored/no secret, mark error controls.

Designer implication

This section should communicate "what context have I volunteered to Jarvis?" It needs clear status and revocation affordances, because trust matters more than configuration density.
