### Settled implementation decisions (build exactly this)

- `extractMaterialSpans(markdown)`: per line → strip heading/quote prefix
  `^\s*(?:[#>]+\s*)+`, then list prefix `^\s*(?:[-*+]|\d{1,3}[.)])\s+`, then all `` [*_`] ``.
  Whitespace-tokenize; strip leading/trailing `[^0-9A-Za-z]` per token; drop empties. Rules:
  token contains digit → material (even at index 0); else index 0 → skipped (sentence-case
  allowance — equivalent to the design's "drop only the FIRST word of a segment-initial run"
  since index ≥1 capitalized words stay material); else `/^[A-Z]/` && length ≥2 → material.
  ASCII-caps heuristic by design — say so in a comment.
- `verifyMarkdownCoverage({markdown, sources, confirmedTexts})`: corpus =
  `[...sources.map(s => s.content), ...confirmedTexts].join("\n").toLowerCase()`; span fails
  if `!corpus.includes(span.toLowerCase())`; dedupe via Set; cap `unverifiedSpans` echo at 64
  (defensive response-size cap; `ok:false` regardless). Returns `{ok, unverifiedSpans}`.

`MATERIAL_CLAIM_KINDS`, `CLAIM_TEXT_MAX_CHARS=500`, `CLAIM_QUOTE_MAX_CHARS=200`,

```ts
const SEGMENT_SPLIT = /[.!?;|]+/; // commas deliberately do NOT split
function normalizePhrase(text: string): string {
  const tokens = text.toLowerCase().match(/[\p{L}\p{N}]+/gu);
  return tokens === null ? "" : tokens.join(" ");
}
export function extractMaterialSegments(markdown: string): { raw: string; phrase: string }[] {
  // per line: strip /^\s*(?:[#>]+\s*)+/ then /^\s*(?:[-*+]|\d{1,3}[.)])\s+/ then /[*_`]/g,
  // split on SEGMENT_SPLIT, trim → raw, normalizePhrase → phrase, skip phrase===""
}
export function verifyMarkdownCoverage(input): MarkdownCoverageVerdict {
  const proposed = extractMaterialSegments(input.markdown);
  if (proposed.length === 0) return { ok: false, unverifiedSpans: [] }; // DEFEAT 2
  // corpus: extractMaterialSegments over each source.content + each confirmedText,
  //   each corpus phrase held as ` ${phrase} ` (space-padded, per-segment — no
  //   cross-line false adjacency)
  // per unique proposed phrase: needle = ` ${phrase} `; unverified unless SOME corpus
  //   string .includes(needle)  ← word-boundary contiguous-run matching
  // unverifiedSpans: echo RAW segment text (cap ~200 chars), slice(0, UNVERIFIED_SPANS_MAX);
  //   truncation must never flip the verdict
}
```

Semantics: a proposed segment passes iff its full normalized token run appears
contiguously inside ONE corpus segment. Sub-phrase of one corpus sentence → pass;
recombination across segments → fail. Unicode tokenization fixes École; whole-segment
matching removes the caps/digit + first-word heuristics entirely.
