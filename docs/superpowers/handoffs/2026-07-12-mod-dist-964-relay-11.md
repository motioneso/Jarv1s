### Two undiagnosed real bugs found this relay — next task

Both surfaced only once install actually started succeeding (earlier relays never got this far).
Root-cause via reading source, same method as the two fixes above — don't guess.

1. **Install never flips to `installed-enabled`.** `boot reconcile accepts the staged download and
   installs the module schema` (test line ~279) and the 0.3.0 update test both get
   `state: "installed-disabled"`, `installedVersion: null` where the test expects
   `"installed-enabled"` / `"0.2.0"`. The schema DOES install now (no more DB errors) but whatever
   phase is supposed to call `setExternalModuleEnabled` (flip disabled→enabled after a successful
   install) isn't running, OR the test's expectation of auto-enable-on-install is wrong and there's
   a missing explicit "accept" step the test needs to call first — **check the plan section for
   Task 7 (boot reconcile phases) and grep `scripts/module-reconcile.ts` for where/whether
   `setExternalModuleEnabled` or an equivalent gets called post-install; compare against what the
   admin route (`routes-module-registry.ts`) does for the same transition** before assuming either
   side is wrong.
2. **Purge fails: `role "jarvis_mod_acme_widgets_runtime" cannot be dropped because some objects
   depend on it`.** Surfaces in `remove+purge then reconcile destroys tables...`, `purge re-run is
   idempotent`, and cascades into `remove keeps data; reinstall resumes...`. Now that install
   actually runs `generateModuleTableRlsSql` (GRANTs + policies to the runtime role), the purge
   path's role-drop doesn't REVOKE those grants / drop the policies first. Grep
   `scripts/module-reconcile.ts` Phase for `"purge"` and whatever drops the runtime role — it
   likely needs a `REVOKE ALL ... FROM <role>` and/or `DROP OWNED BY <role>` before `DROP ROLE`, or
   needs to drop the owned tables (which drops dependent policies/grants) before dropping the role.
   This is a **real pre-existing bug in Task 7's purge phase**, exposed by the same fixture-and-
   install-path fixes that fixed the two bugs already committed this relay — same confidence level.
