- The dashboard is the user-facing place to review, edit, confirm, reject, mark stale/expired,
  supersede, and delete/forget memory records.
- Include pending memory candidates from #529 and active graph memory from #528/#532 in one memory

- Do not build data freshness labels beyond memory state; #541 owns freshness visibility.
