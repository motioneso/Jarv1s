## How to run e2e locally

```bash
pnpm install
pnpm db:up                          # start postgres
pnpm db:migrate                     # apply migrations
pnpm build:web                      # build the web app (e2e uses preview server)
pnpm exec playwright install --with-deps chromium
pnpm test:e2e                       # runs all e2e (or target a file:)
pnpm exec playwright test tests/e2e/onboarding.spec.ts
```

The e2e suite starts a preview server on `127.0.0.1:4173`. Tests use a fresh DB.

## Approach

1. **Investigate each failure.** These are pre-existing (not caused by the PII scrub or chat-stability batch). Some may share a root cause (e.g. the onboarding redirect failures #3/#5 are likely the same bug).
2. **Fix the tests OR the code**, whichever is wrong. If a test asserts outdated behavior (e.g. a UI that was intentionally changed), update the test. If the code is genuinely broken, fix the code.
3. **Run the full e2e suite** to confirm ALL 6 pass + no regressions in the 36 currently-passing tests.
4. **Run the local gate:** `pnpm format:check && pnpm lint && pnpm typecheck && pnpm test:unit` (must stay green).
5. **Commit + push** your branch, open a PR, report back to the Coordinator.

## Guardrails

- **Test-only changes preferred.** If a code fix is needed, keep it minimal and scoped to the test failure.
- **Do NOT touch:** auth, RLS, migrations, secrets, CI workflow files, the PII scrub.
- **Do NOT run `git add -A` or `git add .`** — stage only the files you change.
- **Do NOT touch `docs/coordination/`** — coordinator-only.
- If you discover a failure is actually a real product bug (not a stale test), flag it in your report rather than papering over it.
