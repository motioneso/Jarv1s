## Issue #537 Direction

Spec direction:

- Build on #527 usefulness feedback, #528/#529/#532/#533 memory graph/candidates/confidence/
  dashboard, #534 action permission tiers, #535 long-running goals, #536 scheduled briefings, and
  existing Tasks.
- Define automatic commitment extraction as owner-scoped detection of possible obligations from
  chats, notes, and email, producing reviewable commitment candidates with source links,
  confidence, status, and suggested handling.
- Commitments should distinguish promises, owed replies, deadlines, decisions requiring follow-up,
  and explicit non-actions.
- Do not silently create tasks, reminders, emails, calendar events, or goal updates. Any suggested
  execution must route through #534 and source-owned tools.
- Metadata-only job payloads only: actor/source ids, job kind, command metadata, idempotency key.
  Never private text, prompts, secrets, source bodies, or connector payloads.
- Keep private by default and owner-only in V1. No admin private-data bypass.
- Use source snippets/provenance as evidence. Do not make raw memory records or email/note/chat
  bodies the commitment store.
- Deduplicate and suppress repeated noisy candidates; preserve user rejections.
- Tie person/counterparty identity to #538. Until #538 ships, store bounded labels/source refs only
  and mark ambiguous counterparties clearly.
- Tie audit-log detail to #540 and freshness labels to #541.

## Issue #538 Direction

Spec direction:

- Build on #528 memory entities/aliases, #532 confidence-aware memory, #533 memory dashboard, #537
  commitment candidates, #525 cross-tool reasoning, and #539 future answer provenance.
- Define a unified, owner-scoped person/contact context model for Jarvis, not a full CRM.
- Model people with names, aliases, email addresses or source identities, relationship/context
  notes, confidence/provenance, ambiguity, merge/split behavior, and linked artifacts.
- Source modules remain source-of-truth for emails, calendar attendees, tasks, notes, and memory
  evidence. The person model indexes and links context; it does not copy full source bodies.
- Support queries like "what do I owe Sarah?" by connecting person identities to commitments,
  messages, tasks, notes, goals, and recent interactions.
- Keep private by default and owner-only in V1. No admin private-data bypass.
- Do not build connector sync, an address-book editor, social graph import, or contact-sharing.
- Ambiguous aliases must not auto-merge. Require review or confidence thresholds before merge.
- Tie source-backed explanation/citations to #539, action audit to #540, and freshness to #541.
