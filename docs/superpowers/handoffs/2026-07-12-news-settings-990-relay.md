## Design reasoned through (not yet written to a plan doc)

1. Add `updateNewsTopic(id, input)` to `news-client.ts`, mirroring `deleteNewsTopic`'s shape:
   `requestJson<UpdateNewsTopicResponse>(`/api/news/topics/${encodeURIComponent(id)}`, { method: "PATCH", body: input })`.
2. Rename/recopy the 4 section kickers in `index.tsx` per spec Decision 1: "Publications" / "Topics
   from your publications" / "Publications you add" / "Topics across the web".
3. Extract a new `packages/news/src/settings/describe-topics.tsx` (mirrors `add-source.tsx`) owning:
   one compact add/edit form (topic + optional guidance, Enter submits either input, Edit swaps
   primary action to "Save changes" + exposes Cancel — spec Decision 2), `editingId` state,
   create/update/delete mutations, and local `role="status"`/`role="alert"` announcements (spec
   Decision 5: "Checking topic…"/"Saving changes…" pending copy; "Topic added"/"Changes
   saved"/"Topic removed" success after state visible; preserve inputs + safe-copy alert on failure).
   Receives `customTopics`, `availability`, `needsAttention: boolean`, and a shared `retryRow: () =>
   JSX.Element` render-prop from the parent (parent's `revalidateMutation` state must stay shared
   across "Personalized sources" and "Topics across the web" — both can show "Retry validation").
4. All create/update/delete success paths invalidate both `newsQueryKeys.personalization` and
   `newsQueryKeys.overview` (spec Decision 6).
5. Empty-topics state: honest copy ("News still uses your selected publications") + prominent add
   form, not a dead end (spec Decision 7). Missing-prereq gate stays as-is (`PrereqGate`).
6. New CSS classes in `news-settings.css` for edit-mode form state + status/alert regions — colors
   via `var(--token)` only, no raw hex/rgb (Hard Invariant).
7. Extend `tests/unit/news-settings-pane.test.tsx`: new fixtures/assertions for section
   rename/recopy, empty-topics honest state, edit-mode load/cancel, safe-text rendering of hostile
   Edit-affordance labels (must not break the existing adversarial-content suite), existing
   `topicCreateErrorMessage()` coverage untouched.
8. New `tests/e2e/news-settings.spec.ts` from scratch: local stateful route mock (inline in this
   spec file, NOT a shared `mock-*.ts` helper — spec is explicit about this) covering
   personalization GET, topic POST/PATCH/DELETE round-trips, revalidation POST. Navigate via
   `/settings?section=modules&module=news`. No live web-search/model/RSS/worker.
