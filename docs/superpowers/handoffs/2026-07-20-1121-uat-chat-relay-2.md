## Approved plan shape (Option A, from Ben via the original handoff doc)

Dedicated test/service Anthropic account's `claude setup-token` output, held host-side
GPG-encrypted, injected via the existing compose `--env-file` convention into an ephemeral
per-run UAT stack, written into the claude-scoped cli-auth volume via the EXISTING
`persistProviderToken` contract, driving the EXISTING `needs_login → ready` login-service
transition + `autoRegister.ensureDefaultChatModel` — behind an explicit opt-in mode gate that
**fails closed** (refuses to start, no silent fallback) when the credential is absent. Default/CI
credential-free path and #1025 fake-provider isolation stay completely untouched.

**Hard invariant, non-negotiable:** the write→confirm floor. No mode, including real-chat opt-in,
may ever let a cookie-authed direct tool invocation bypass `action_request` →
Playwright-click-Approve → gateway-executes. Direct invoke must always return
`403 confirmation_required`; `/resolve` stays audit-only.
