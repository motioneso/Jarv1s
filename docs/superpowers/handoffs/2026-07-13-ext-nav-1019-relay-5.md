recreate unless needed — just re-derive with curl/browser devtools if you need to re-check):

  array — only ~22 built-in modules. `GET /api/me/modules` likewise omits it.

`requireManifests()` in `packages/settings/src/routes-modules.ts` (read around its GET
`/api/me/modules` handler, lines ~250-274, and the `/api/modules` handler) — trace where it
sources its manifest list and whether/how it's supposed to merge in `discoverExternalModules()`'s
boot-time result. This is either a real #1019 implementation gap (route doesn't merge dynamically
discovered external modules) or a UAT env/wiring issue on my part — not yet distinguished.

Frontend nav logic itself (`apps/web/src/app-route-metadata.ts` `buildShellNavigation` /
`MODULES_SECTION`, `apps/web/src/shell/app-shell.tsx`) is confirmed correct/already implemented
(commits `75c3a596`/`9fa0b203`) — it just never receives a `module.external === true` entry to
render, because the module never appears in the API response to begin with.
