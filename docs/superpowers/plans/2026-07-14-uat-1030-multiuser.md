# UAT Multi-User Seed Implementation Plan

> **For coordinated build agents:** REQUIRED SUB-SKILL: Use `superpowers:test-driven-development` and execute this plan inline, task by task. `coordinated-build` disables plan-execution subagents in this repo.

**Goal:** Replace the `multi-user` UAT seed stub with two loginable owners, per-owner lived-in data, and an adversarially tested owner-or-share task fixture.

**Architecture:** Generalize the existing privileged account bootstrap just enough to create a fixed non-admin owner while preserving `seedSoloAdmin`. Reuse the existing `app_runtime` runner and `ADMIN_DATA_CHUNKS` for both owners, then grant one admin-owned task to owner 2 through `SharesRepository.grant` under owner 1's `AccessContext`. Keep the existing runner `finally` as the only pool-cleanup path.

**Tech Stack:** TypeScript, Kysely, PostgreSQL FORCE RLS, Vitest, pnpm.

## Global Constraints

- No `BYPASSRLS`, RLS-policy changes, migrations, raw share inserts, raw Kysely root repository access, or raw `fs`.
- `AccessContext` remains `{ actorUserId, requestId }`; owner 2 data is written only with owner 2's `actorUserId`.
- Use fixed IDs and `UAT_SEED_BASE_TIMESTAMP`; do not add `Date.now()` or `new Date()` to seed code.
- Preserve spec-derived `uatLevel`, idempotent chunk behavior, and `runner.destroy()` in the existing `finally` on success and failure.
- Keep `job-search` owner-1-only: it is instance-level admin configuration, not per-owner lived-in data. Owner 2 receives every per-owner `ADMIN_DATA_CHUNKS` seeder (`news`, `sports`, `tasks`, `calendar`, `notes`) that is not excluded.
- Stage explicit paths only; never edit `docs/coordination/`.
