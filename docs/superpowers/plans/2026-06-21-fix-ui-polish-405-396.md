### Task 1: #405 — Crisp dark-theme background texture

**Files:**

- Modify: `apps/web/src/styles.css:44` (body background-image rule)

**Interfaces:**

- Consumes: existing SVG data-URI inline noise texture on `body`
- Produces: same texture rendered at native 128 px tile size, tiling via `repeat`

**Root cause:** The `body` selector sets `background-image` with a `viewBox='0 0 200 200'` SVG noise tile but no `background-size` or `background-repeat`. The single tile stretches across the whole viewport → upscaled blur that reads as a failed load.

**Fix:** Add `background-size: 128px` + `background-repeat: repeat` directly after `background-image` in the body rule. The existing opacity is `0.03` (the SVG rect), which is in the whisper range — leave it unchanged.

### Task 2: #396 — Wellness chart: default scroll to today on mobile

**Files:**

- Modify: `apps/web/src/wellness/wellness-chart.tsx` (add `useLayoutEffect` import + effect)

**Interfaces:**

- Consumes: existing `containerRef` (already attached to `.wl-chart__plot` div at line 150); existing `days` prop array
- Produces: on mount and when `days.length` changes, `containerRef.current.scrollLeft` is set to `scrollWidth` so the right edge (today) is visible

**Note on location:** The handoff suggested adding scroll in `wellness-trends.tsx`, but the actual scroll container (`.wl-chart__plot`, styled `overflow-x: auto` at mobile widths) is rendered inside `WellnessChart` with `containerRef` already on it. The correct location is `wellness-chart.tsx` — adding a `useLayoutEffect` there reuses the existing ref without adding a wrapper div.

**Why `useLayoutEffect` not `useEffect`:** Layout effect fires synchronously after DOM mutations, before the browser paints — prevents a visible flash of the left-anchored position.

**Why trigger on `days.length`:** The effect sets initial scroll position only; it does NOT re-fire on every render (so it doesn't fight the user scrolling left). Triggering on `days.length` handles the case where range changes (14 → 30 days) — the new range's right edge is again the default.


Change to:

```tsx
import { useState, useRef, useEffect, useLayoutEffect } from "react";
```
